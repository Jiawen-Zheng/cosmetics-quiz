## 修复的问题汇总

### 第一轮修复：配置文件问题
1. **移除无效配置**
   - 文件：`next.config.ts`
   - 问题：`reactCompiler` 配置项在 Next.js 15.1.4 中不被识别
   - 解决：移除该配置项

2. **修复 PostCSS 配置**
   - 文件：`postcss.config.mjs`
   - 问题：使用了 Tailwind v4 语法但项目是 v3
   - 解决：改用标准的 `tailwindcss` + `autoprefixer` 配置

3. **添加 Tailwind 配置**
   - 文件：`tailwind.config.ts`（新建）
   - 问题：缺少 Tailwind 配置文件
   - 解决：创建标准的 Tailwind v3 配置文件

4. **修复 CSS 导入**
   - 文件：`src/app/globals.css`
   - 问题：使用了 Tailwind v4 的 `@import "tailwindcss"` 语法
   - 解决：改用 v3 的 `@tailwind` 指令

### 第二轮修复：ESLint 和 Suspense 问题
5. **修复 ESLint 配置**
   - 文件：`eslint.config.mjs`
   - 问题：使用了不兼容的 API
   - 解决：使用 `@eslint/eslintrc` 的 `FlatCompat`

6. **添加 Suspense 边界**
   - 文件：`src/app/result/page.tsx`
   - 问题：`useSearchParams()` 需要 Suspense 包装
   - 解决：将组件拆分并用 `<Suspense>` 包装

### 第三轮修复：代码 Lint 错误
7. **转义特殊字符**
   - 文件：`src/app/copyright/page.tsx`
   - 问题：HTML 中的引号和撇号未转义
   - 解决：使用 HTML 实体（`&ldquo;`、`&apos;` 等）

8. **使用 Next.js 组件**
   - 文件：`src/app/copyright/page.tsx`、`src/app/quiz/page.tsx`
   - 问题：使用了原生 `<a>` 和 `<img>` 标签
   - 解决：改用 Next.js 的 `<Link>` 和 `<Image>` 组件

这是**正常的构建流程**，问题是逐层暴露的：

```
第一层：配置文件错误
   ↓ 修复后
第二层：ESLint/Suspense 配置问题
   ↓ 修复后
第三层：代码 Lint 错误
   ↓ 修复后
✅ 构建成功
```

每修复一层问题，下一层问题才会显现。这不是第一个 bug 没解决，而是构建系统的正常工作方式。
