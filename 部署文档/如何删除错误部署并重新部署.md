# 如何删除 Vercel 错误部署并重新部署

## 方法一：在 Vercel Dashboard 删除失败的部署

### 步骤：

1. **进入项目页面**
   - 登录 Vercel Dashboard
   - 选择您的项目 `cosmetics-quiz`

2. **查看部署列表**
   - 点击顶部的 "Deployments" 标签
   - 您会看到所有部署记录

3. **删除失败的部署**
   - 找到状态为 "Failed" 或 "Error" 的部署
   - 点击部署记录进入详情页
   - 点击右上角的 "..." 菜单
   - 选择 "Delete Deployment"
   - 确认删除

4. **重复操作**
   - 对每个失败的部署重复上述步骤

## 方法二：触发新的部署（推荐）

**不需要删除旧部署**，只需触发一个新的成功部署即可：

### 选项 A：通过 Git 推送触发

```bash
cd cosmetics-quiz

# 创建一个空提交来触发部署
git commit --allow-empty -m "chore: 触发重新部署"

# 推送到 GitHub
git push
```

### 选项 B：在 Vercel Dashboard 手动触发

1. 进入项目页面
2. 点击 "Deployments" 标签
3. 找到最新的成功提交（commit: 3ef1fda）
4. 点击右侧的 "..." 菜单
5. 选择 "Redeploy"
6. 确认重新部署

### 选项 C：通过 Vercel CLI 触发

```bash
# 安装 Vercel CLI（如果还没安装）
npm install -g vercel

# 登录
vercel login

# 进入项目目录
cd cosmetics-quiz

# 触发部署
vercel --prod
```

## 为什么不需要删除旧部署？

1. **不影响当前部署**：失败的部署不会影响您的生产环境
2. **保留历史记录**：可以查看之前的错误日志，方便排查问题
3. **自动清理**：Vercel 会自动管理旧部署，不会占用太多资源
